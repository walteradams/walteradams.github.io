Greedy Peg

HTML5, CSS3, Responsive, JavaScript

VIEW WEBSITE

As part of the brief I had to avoid the use of AJAX as much as possible to prevent additional content being inaccessible in the event of a connection being lost. Despite this, we still wanted to provide the feeling of a dynamic app. Much of the secondary or additional content is delivered with the initial request in JSON which can then later be loaded into the page where required, for example, when loading more destinations. We kept the JSON minimal so as not to load too much data in the initial request yet still provide enough meaningful secondary content. This also helped keep the design simple and the layout quick to navigate as additional content is hidden away until required.

A lot of Greedy Peg’s information can be accessed via maps which I implemented using the Google Maps API. As with additional data on content pages, map data is also loaded via JSON delivered with the initial page request. The JSON includes data such as map location, marker description and title, as well as settings that allow Greedy Peg content creators to change how the map looks and behaves; giving them away to change marker colours, type of map such as; article, news, and so on. Development of the Google Maps API included use of custom markers, marker clustering to prevent too many markers in one area at the same time (not in use at time of writing), and tooltips.

As part of the design, a slide-out “tray” needed to be implemented where more content is loaded such as external articles, minimal cached external articles (giving the user a way to read referenced pages without the weight of lots of styles, JavaScript, and so on), and a photo/video gallery. The tray loads this content via an iframe over the top of the current page. In the event the user is experiencing a poor internet connection and the content doesn’t load, they can easily go back to the page underneath the tray to at least have access to the information they were previously reading until a better connection is available. I developed the tray to be automatically customized based on the type of link it is loading, for example, loading a menu specific to the map or showing a back button when navigating from a Greedy Peg hosted page to an external resource.

Conclusion

When working on front-end development projects I am always considerate of the fact that user experience is not only improved by lighter-weight experiences but in some circumstances, may actually be cheaper for the user when they are using a data plan that charges based on amount downloaded. Working on Greedy Peg gave me the opportunity to put this to the test and the client reports that the site has been received very well and works reliably over what is typically a poor connection. I look forward to continue working with Greedy Peg as the website and company grows.
